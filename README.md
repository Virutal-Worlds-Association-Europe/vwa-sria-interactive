# Virtual Worlds SRIA Portal

A static documentation website for the **Strategic Research & Innovation Agenda (SRIA) for Virtual Worlds**, built with Docusaurus v3. This portal transforms a comprehensive PDF research document into an accessible, navigable, and searchable web experience for policymakers and researchers.

## 🎯 Overview

The Virtual Worlds SRIA Portal is a modern documentation website that presents Europe's strategic research agenda for virtual worlds technology. The portal covers four major areas:

1. **Applications & Opportunities** - Industry, Healthcare, Media, Arts, Education, Security, and Cities
2. **Technical Outlook** - Visualization, Interaction, Authoring, Standards, Digital Twins, and AI
3. **Socio-Economic Outlook** - Human Rights, Governance, Economics, Sustainability, and Trust
4. **European Context** - Data, Infrastructure, and Optics/Photonics

## ✨ Features

### Core Functionality

- **📚 21 Comprehensive Sections** - All chapters from the original SRIA document
- **🔍 Local Search** - Fast, privacy-respecting search with CMDK interface
- **💬 Context-Aware Feedback** - Integrated Tally.so forms with automatic section detection
- **🎨 Dark Mode** - Fully supported light and dark themes
- **📱 Responsive Design** - Optimized for mobile, tablet, and desktop
- **♿ Accessible** - WCAG compliant with keyboard navigation and screen reader support
- **🌍 GDPR Compliant** - No tracking cookies, no external data transfers

### Technical Highlights

- Built with **Docusaurus v3** and **TypeScript**
- **Local search** - No external dependencies or API calls
- **Static site generation** - Fast loading, easy hosting
- **Automatic sidebar generation** - Based on folder structure
- **Customizable theming** - CSS variables for easy branding

## 🚀 Quick Start

### Prerequisites

- **Node.js** 18.0 or higher
- **npm** 9.0 or higher

### Installation

```bash
# Clone the repository
git clone https://github.com/Virutal-Worlds-Association-Europe/vwa-sria-interactive.git
cd vwa-sria-interactive

# Navigate to the portal directory
cd sria-portal

# Install dependencies
npm install

# Start development server
npm start
```

The site will open at `http://localhost:3000`.

### Building for Production

```bash
# Create production build
npm run build

# Test production build locally
npm run serve
```

## 💻 Development

### Available Commands

```bash
# Development
npm start              # Start dev server (http://localhost:3000)
npm run typecheck      # TypeScript type checking
npm run clear          # Clear cache

# Building
npm run build          # Create production build
npm run serve          # Serve production build locally

# Quality
npm run lint           # Run linter (if configured)
npm run format         # Format code (if configured)
```

### Configuration Files

- **`docusaurus.config.ts`** - Main site configuration
- **`sidebars.ts`** - Sidebar structure (autogenerated)
- **`tsconfig.json`** - TypeScript configuration
- **`package.json`** - Dependencies and scripts

### Adding Content

1. Create a new `.md` file in the appropriate `/docs` subdirectory
2. Follow the naming convention: `{chapter}{letter}-{name}.md`
3. Add frontmatter (optional):
   ```yaml
   ---
   title: Your Title
   description: Your description
   ---
   ```
4. Content will automatically appear in the sidebar

### Customizing Theme

Edit `src/css/custom.css` to customize:
- Colors (CSS variables)
- Fonts
- Spacing
- Component styles
